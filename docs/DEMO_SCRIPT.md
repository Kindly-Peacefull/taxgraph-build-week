# TaxGraph demo script (under three minutes)

This script must be updated after the final feature verification. It currently describes the intended verified MVP path only.

**0:00–0:20 — Problem.** A product sold as one “AI assistant” can combine automated software, implementation, support, and training. TaxGraph maps the questions before a cross-border sale instead of pretending one rate answers everything.

**0:20–0:40 — Load the mixed product.** Open TaxGraph and load the France B2C demo. Point out the Serbian seller, consumer customer, and optional contract excerpt.

**0:40–1:00 — Decomposition and contract facts.** Show the service components and the known, inferred, missing, and conflicting fact groups with provenance.

**1:00–1:25 — France B2C touchpoints.** Open Tax Touchpoints. Show the preliminary ESS, destination, evidence, and reporting-route rows. Emphasize that every legal claim is pending human review.

**1:25–1:55 — Compare Germany B2B.** Open Scenario Comparison. Show the changed customer facts, affected rules, German B2B place-of-supply path, reverse-charge checklist, and professional-review flags.

**1:55–2:15 — Answer a missing fact.** Return to Overview, answer the location-evidence question, and show the rule engine rerun and changed touchpoint highlight without another GPT call.

**2:15–2:35 — Open evidence.** Open a source footnote to display its exact official excerpt. Show the clearly labelled Germany VIES fixture and audit timestamp.

**2:35–2:55 — Explain the architecture.** GPT-5.6 normalizes and decomposes on the server. Typed code owns R1–R12, source gates, citations, diffs, and reruns. Fixture mode uses the same schemas and engine.

**2:55–3:00 — Close.** TaxGraph turns a vague cross-border AI sale into a reviewable pre-sale evidence map.
